# 历史记录与 Diff — 需求规格

## 概述

记录最近查看/粘贴过的 JSON（**默认关闭**，用户手动开启），并支持两份 JSON 左右对比，解决"刚才那个响应变了什么"。

## 项目信息

- 项目名: json-keeper
- 架构类型: 单体前端应用（Chrome MV3 扩展 · 零依赖原生 JS · 无构建）

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-07-16 | v1 | 初始需求（源: `specs/docs/PRD-json-keeper-ui.md` §5.3） |

## 用户故事

- 作为**反复调同一个接口的开发者**，我想要对比这次和上次的响应，以便我一眼看出改了什么。
- 作为**隐私敏感的用户**，我想要历史记录默认不开、并能随时清空，以便我看过的带 token 的响应不会不知不觉留在盘上。
- 作为**手上有两份 JSON 的人**，我想要直接粘贴/拖入第二份对比，以便我不必先把它们存成文件再找 diff 工具。

## 功能需求

1. [F-301] 历史列表：时间 + 大小 + 首行摘要，存 `chrome.storage.local`。
2. [F-302] popup 显示最近 3 条；查看器内可开完整历史面板。
3. [F-303] 单条删除 + 一键清空。
4. [F-304] 容量上限（条数与总字节），超出淘汰最旧，UI 明确说明规则。
5. [F-305] 两份 JSON 左右分栏对比，标出新增 / 删除 / 修改。
6. [F-306] Diff 三个入口：从历史选两条 · 粘贴第二份 · 拖入 `.json` 文件。
7. [F-307] Diff 空结果态：两份完全相同时明确说"无差异"，不是空白。
8. [F-308] **历史默认关闭**；首次使用处提供开启入口，并说明"数据仅存本地"。

## 非功能需求

- **性能**: Diff 对 1MB 以内 JSON 在 1s 内出结果。
- **安全**: 历史内容是用户最敏感的数据（生产 API 响应常含 token）—— **默认不写盘**；开启后仅存 `chrome.storage.local`，**禁止任何外发**；渲染一律 `esc()`。
- **兼容性**: 零依赖 —— diff 算法自研，不得引入 npm 包或 CDN。
- **体积**: 新增 `history.js` / `jsondiff.js` 各 ≤400 行。

## 验收标准

- [ ] [AC-301] 全新安装后打开若干 JSON → `chrome.storage.local` 中**无任何历史条目**（默认关闭生效）。
- [ ] [AC-302] 手动开启历史后打开 JSON → 出现一条记录，含时间、大小、首行摘要。
- [ ] [AC-303] popup 显示最近 3 条；点击任一条直接载入 viewer。
- [ ] [AC-304] 单条删除后该条消失；一键清空后列表为空且 storage 中对应键被清除。
- [ ] [AC-305] 写入超过上限（条数或总字节）→ 最旧的被淘汰，且 UI 有规则说明文案。
- [ ] [AC-306] 对比 `{"a":1,"b":2}` 与 `{"a":9,"c":3}` → 标出 `a` 修改（1→9）、`b` 删除、`c` 新增。
- [ ] [AC-307] 对比两份完全相同的 JSON → 显示"无差异"说明文字，不是空白面板。
- [ ] [AC-308] 三个 Diff 入口均可用：历史选两条 / 粘贴第二份 / 拖入 .json 文件。
- [ ] [AC-309] 对比含大整数的两份 JSON（`136986234663732436` vs `136986234663732437`）→ 正确识别为"修改"，且两侧显示均不失真。
- [ ] [AC-310] 历史条目含 `<script>` 时，列表以纯文本显示摘要，无弹窗。
- [ ] [AC-311] `grep -rnE "fetch\(|XMLHttpRequest" history.js jsondiff.js` 无命中（零网络红线）。
- [ ] [AC-312] 复跑防护网基线（`node --test tests/`）全绿。

## 开放问题

- 已全部确认：历史 = **默认关闭，用户手动开**（人工确认，理由: 用户看的 JSON 常含生产 token 与个人信息，默认存盘与"no telemetry"的气质冲突）；Diff 入口 = **三个全做**（历史选两条 / 粘贴第二份 / 拖入文件，人工确认）。
- **已知张力**（不阻塞，但需在 UI 上化解）：历史默认关 → "从历史选两条对比"这个入口在默认状态下是空的，需引导用户先开启历史。见 design.md「模块 4」。
- **无设计稿**：Stitch 的 `s3-table-diff-dark` 含 Diff 视图的**部分**结构（左栏 Tree/History/Diff/Compare、`Table View | Diff View` 切换），可作结构参考；历史面板与三个入口无稿，由前端按 design.md 实现。
