# 查询与表格视图 — 需求规格

## 概述

新增 JSONPath 查询栏（筛出子集）与表格视图（数组型 JSON 用表格看），并把"缺失字段"显式标出而非留空。

## 项目信息

- 项目名: json-keeper
- 架构类型: 单体前端应用（Chrome MV3 扩展 · 零依赖原生 JS · 无构建）

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-07-16 | v1 | 初始需求（源: `specs/docs/PRD-json-keeper-ui.md` §5.1） |

## 用户故事

- 作为**调接口的开发者**，我想要用 `$.users[*].email` 筛出我要的字段，以便我不用在几千行里靠搜索高亮找。
- 作为**看列表型 API 响应的人**，我想要用表格看数组，以便我一眼比较各条记录的同名字段。
- 作为**核对数据的人**，我想要清楚看出某条记录是"字段值为 null"还是"根本没有这个字段"，以便我不会误判接口行为。

## 功能需求

1. [F-101] 查询栏：输入 JSONPath 表达式，结果替换主视图显示匹配子集。
2. [F-102] 结果计数（`3 matches`）+ 回到完整视图的明确出口。
3. [F-103] 查询语法错误态：就地提示错在哪，**不清空已有结果**。
4. [F-104] 表格视图：列 = 各元素 key 的并集，行 = 数组元素；**缺失字段显式标记**，与值为 `null` 可区分。
5. [F-105] 表格 ↔ 树互跳：点表格单元格跳到树中对应节点。
6. [F-106] 表格不可用态：非数组 / 元素非对象时，分段器该项置灰并说明原因。
7. [F-107] 表格内嵌套对象显示为 `{…}`，可点开查看子树（不拍平成列）。

## 非功能需求

- **性能**: 表格对 1000 行以内数组即时渲染；超出走现有大文件降级路径。
- **安全**: 表格单元格内容为不可信 JSON，一律 `esc()`；查询表达式**禁止**用 `eval` / `new Function` 求值（`.claude/rules/security.md` 红线）。
- **兼容性**: 零依赖 —— JSONPath 求值器自研，不得引入 npm 包或 CDN。
- **体积**: 新增 `jsonpath.js` 与 `table.js` 各 ≤400 行。

## 验收标准

- [ ] [AC-101] 对 `{"users":[{"email":"a@b.c"},{"email":"d@e.f"}]}` 查询 `$.users[*].email` → 结果显示 2 条，计数为 `2 matches`。
- [ ] [AC-102] 查询后存在可点击的"清除查询/回到完整视图"入口，点击后恢复完整树。
- [ ] [AC-103] 输入非法表达式 `$.users[`：出现语法错误提示，**上一次的查询结果仍在屏幕上**，未被清空。
- [ ] [AC-104] 对 `[{"a":1,"b":2},{"a":3}]` 切表格：列为 `a`、`b`；第二行 `b` 列显示"缺失"标记，与值为 `null` 的渲染**在视觉上可区分**。
- [ ] [AC-105] 点击表格任一单元格 → 切到树视图且对应节点被定位并高亮。
- [ ] [AC-106] 打开非数组 JSON（如 `{"a":1}`）：`Table` 分段置灰，hover 说明原因（如"仅数组可用表格视图"）。
- [ ] [AC-107] 对 `[{"user":{"city":"SH"}}]` 切表格：`user` 列显示 `{…}`，点击后可见子树；列头**不出现** `user.city`。
- [ ] [AC-108] 查询表达式不经 `eval`/`new Function` 求值 —— `grep -nE "eval\(|new Function" jsonpath.js` 无命中。
- [ ] [AC-109] 喂含 `<script>` 的字段值切表格：单元格以纯文本显示，无弹窗。
- [ ] [AC-110] 复跑防护网基线（`node --test tests/`）全绿。

## 开放问题

- 已全部确认：查询语法 = **JSONPath**（人工确认，jq 因自研引擎工作量与学习成本被否）；嵌套对象 = **单元格内 `{…}` 可点开**（人工确认，不拍平成列）。
- 遗留（不阻塞）：JSONPath 支持的语法子集范围见 design.md「技术决策」，首版不实现 filter 表达式（`?()`）。
