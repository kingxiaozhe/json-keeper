## 核对挖出的缺口

### 缺口 1〔真洞 · 已修〕viewer 页的 Format 是个哑巴

**没有任何任务覆盖这个面**：T-007 加了 viewer 页粘贴区，T-008 的任务书写的是「**popup** 三态」。

空输入点 Format → `out.innerHTML = ""` → **什么都不发生、零解释**。这正是 T-008 用一整个任务消灭的那条差评（被替代品 44 条差评的头号来源），**在另一个粘贴入口原样活着** —— 而且它就是 popup.js 注释里称作"修复路"的那个地方：popup 把带着坏 JSON 的人送过去，那边的按钮却是哑的。

**修**：`util.js` 加 `guardEmpty(input, btn)` + `EMPTY_HINT`，popup 与 viewer **共用一份**（两个面的拒绝措辞必须一致 —— 它们曾各写一份主题代码，token 在一个版本内就漂了）。viewer 页三处同步：`input` 事件、**popup 交接后**（赋 `value` 不触发 `input`，不补这一处的话，从 popup 过来的人会看到一个装满 JSON 却禁用着的按钮）、初始。
**网**：`tests/viewerpage.test.mjs` 8 条；6/6 变异全抓（含"两个面措辞漂移"）。

### 缺口 2〔真洞 · 已修〕两处写死的颜色，其中一处**根本没有深色版**

AC-001 说的是"深浅**两版**"。渲染出来长什么样 Node 里验不了，但**"这个颜色在两版里都被想过吗"是静态可核的** —— 一查就露馅（对比度按 WCAG 相对亮度算的，不是看的）：

| 元素 | 颜色 | 浅 panel | 浅 canvas | 深 canvas | 深色版 |
| --- | --- | --- | --- | --- | --- |
| `.jk-error` 解析错误 | `#d6453c` | 4.40 ✗ | 3.85 ✗ | 4.12 ✗ | **压根没有** |
| `.jk-warn` 重复 key 警告 | `#b8860b` | **3.25 ✗** | 2.85 ✗ | — | 有（`#e3b341`，9.13 ✓） |

（AA 正文要 4.5:1）

两条都不是小事：`.jk-error` 是**你最需要读清的那一行**，而三种底色下同一个红、深色下没人管过；`.jk-warn` 是**重复 key 警告 —— 护城河本身**，在浅色主题里 3.25:1，比它旁边的 `● valid JSON`（`--jk-good`，走 token、两版都选过）差得远。

**修**：按 `--jk-good` 的样板补 `--jk-bad`（浅 `#c62828` 5.62/4.92 ✓，深 `#ff7b72` 7.05/7.71 ✓）与 `--jk-warn`（浅 `#8a6100` 5.54/4.85 ✓，深 `#e3b341` 沿用）到**三层**主题里，两处改走 token。连带删掉 `.jk-warn` 那两行手写的深色覆盖 —— **那正是 T-005 建 tokens.css 要消灭的重复**。
