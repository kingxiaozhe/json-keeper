# UI Shell 重做 — 需求规格

## 概述

重做 JSON Keeper 三个界面的视觉与信息架构，补齐现状缺失的空态/错误态/加载态，并把 `core.js` 拆分为可容纳后续功能的模块结构。

## 项目信息

- 项目名: json-keeper
- 架构类型: 单体前端应用（Chrome MV3 扩展 · 零依赖原生 JS · 无构建步骤）

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-07-16 | v1 | 初始需求（源: `specs/docs/PRD-json-keeper-ui.md` §2–§4、§6） |

## 用户故事

- 作为**打开 JSON 网址的开发者**，我想要一眼看出这份 JSON 是否可信（大整数是否保真、有无重复 key），以便我不用怀疑自己看到的数据。
- 作为**第一次用这个扩展的人**，我想要在 popup 里粘贴就有反馈，以便我不会像用 JSONVue 那样"不知道怎么用"。
- 作为**贴错了内容的用户**，我想要就地知道哪里错了，以便我不用白跳一个标签页才看到报错。
- 作为**打开 20MB JSON 的人**，我想要知道它正在处理而不是死了，以便我不会直接关掉标签页。

## 功能需求

1. [F-001] 三个界面（S1 接管页 / S2 popup / S3 viewer 页）按设计基准重做视觉，深浅双主题各一版。
2. [F-002] S2 popup 严格 328px 宽（Chrome popup 物理约束）。
3. [F-003] 工具栏信息架构重构：现有 11 组控件 + 后续 4 组新功能需可扩展地组织，不得一字排开。
4. [F-004] 三个信任信号存在感不弱于现状：big-int 徽章、重复 key 警告、状态栏 `big integers kept exact · no ads · no telemetry`。
5. [F-005] S2 空输入态：按钮禁用或就地提示，消除"点了没反应"。
6. [F-006] S2 非法 JSON 就地错误态：输入框警示描边 + 一行错误摘要，不跳转。
7. [F-007] S2 超长输入提示：将以大文件模式打开。
8. [F-008] S1/S3 建树加载态：大文件切 Pretty 时有进度或骨架反馈。
9. [F-009] S1/S3 空对象 `{}` / 空数组 `[]` 的明确空态。
10. [F-010] S1/S3 搜索无结果态：明确说明，不是"全行变暗像坏了"。
11. [F-011] S1/S3 大文件降级理由可见，并提供"仍要构建树"的明确入口。
12. [F-012] S3 viewer 页粘贴输入区独立于 S1，`⌘/Ctrl+Enter` 渲染。
13. [F-013] `core.js` 拆分为多个 ≤400 行的模块，为后续 feature 留出容纳空间。

## 非功能需求

- **性能**: 树构建对 1MB 以下 JSON 保持当前的即时渲染；大文件懒构建行为不得退化。
- **安全**: 所有用户数据经 `esc()`/`escAttr()` 转义；新增任何 `innerHTML` 拼接必须转义；禁止引入 `eval`/`new Function`/远程脚本（`.claude/rules/security.md`）。
- **兼容性**: 仅 Chrome/Chromium MV3；不引入构建步骤、不引入 npm 依赖、不引入远程 CDN 资源（`.claude/rules/coding-style.md`）。
- **体积**: 拆分后单文件 ≤400 行、单函数 ≤40 行。

## 验收标准

- [ ] [AC-001] 三个界面深浅两版共 6 个状态，逐页对照 `design-baseline/` 的元素清单核对，元素无缺失（结构基准，非像素）。
- [ ] [AC-002] popup 在 Chrome 中实测渲染宽度 = 328px。
- [ ] [AC-003] 空输入点 `Format JSON →`：按钮为禁用态，或出现提示文案；不存在"无任何反应"。
- [ ] [AC-004] 输入 `{"a":}`：popup 内出现错误提示且**不打开新标签页**。
- [ ] [AC-005] 打开 `{}`：出现空态说明文字，不是空白树。
- [ ] [AC-006] 搜索 `zzzz`（无命中）：出现"无结果"说明，不是全部行变暗。
- [ ] [AC-007] 打开 > 1MB JSON：状态栏出现降级说明，且存在可点击的"构建树"入口；点击后出现加载反馈。
- [ ] [AC-008] 状态栏在所有状态下常驻 `big integers kept exact · no ads · no telemetry`。
- [ ] [AC-009] 喂 `{"id":136986234663732436}`：徽章显示 `✓ 1 big-ints exact`，Copy 出的文本粘回仍为原值。
- [ ] [AC-010] 喂 `{"a":1,"a":2}`：出现重复 key 警告，显示值为 2。
- [ ] [AC-011] 喂 `{"<img src=x onerror=alert(1)>":"<script>alert(1)</script>"}`：无弹窗，恶意串以纯文本显示（含 title 属性内）。
- [ ] [AC-012] 访问普通 HTML 页（example.com）：页面完全不变。
- [ ] [AC-013] `wc -l *.js` 结果中无文件 > 400 行。
- [ ] [AC-014] 全部手动冒烟清单（`.claude/rules/testing.md` 10 条）通过。
