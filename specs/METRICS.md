### T-004 详情

- **产出**：三段式工具栏 + `⋯` 溢出菜单 + `addMenuItem` 契约（后 4 个 feature 的唯一挂载入口）
- **一个分歧按证据定**：design 草图写 `Tree|Table|Raw|Min`，但设计稿实物是 `Raw|Pretty|Table|Min`（`Tree` 是它左栏导航条目，feature 4 已明确不采用那套左栏），README 与 storage 值也都是 `pretty` → **保留 Pretty**
- **自查抓到一个我引入的泄漏**：viewer 页每次 Format 都调 mountViewer，菜单往 document 挂永不移除的 click 监听器 → 实测 5 次 render 累积 5 个（反复粘贴调试正是 viewer 页主场景）。已修 + 入网
- **审查拦截 6 条**（独立选靶，换肤半边 **7/7 存活**）：
  1. **回归**：`addMenuItem` 无 `visible:false` 默认 → Raw/Min 下 Collapse all 是死键，点了还把标签翻成"Expand all"对着全展开的树撒谎。**我自己的注释正好承诺了相反的事**
  2. **信任回归**：Sort 的唯一指示器被收进关着的菜单，而 `jk:sort` 跨会话持久化 → 排序一次后每个 JSON 都被重排，`Copy JSON` 静默给出重排结果、界面零提示。与"不偷改你的数据"直接冲突
  3. **L-009 第四次重演**：换肤是唯一从 innerHTML（幻影）搬进 createElement（可测）的控件，免死金牌到期却零测试
  4. **L-011 复发**：两条测试标题撒谎（"有且仅有五项"实断言四项；"onClick 会跑"从没点击过）
  5. `role="menu"`/`menuitem` 是空头支票，且 `<select>` 当 menu 子节点是**结构违规** → 摘掉
  6. `escAttr(v)` 是 cargo cult —— 转了属性位的 `v` 却没转文本位的 `t`，**不一致本身就证明是抄的**
- **裁决（第 2 条，产品级）**：保留 sort 功能与持久化，但把状态提到**常驻区**（`⇅ A→Z` 徽章 + tooltip 明说"Copy 给的是这个顺序"）。理由：藏在关着的菜单后面等于静默改数据
- **修复后**：195 测试；审查的 7 靶 6/7 被抓，第 7 个（tooltip 丢失）**按 L-008 不补测试** —— 视觉细节归冒烟清单，为它写断言是形式主义
